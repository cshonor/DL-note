# 《深度学习图解》第3章 · 3.9 多输入多输出与向量–矩阵乘法

## 3.9 多个输入且多个输出：权重矩阵与 `vect_mat_mul`

> **说明：** 本篇**第七节**是全章「**权重矩阵在左、输入列向量在右**（**y = W·x**）」与框架写法的集中对照；其他小节凡涉及点积 / `vect_mat_mul` 处，与此约定一致。

### 一、本节定位

在 **多输入单输出**（点积求和）与 **单输入多输出**（标量与向量逐元素乘）之后，把两者合起来：**输入仍是特征向量，输出是多个预测值**，权重矩阵**每一行**与输入做一次点积，得到对应输出的一个分量。与教科书里 **「权重矩阵在左、输入列向量在右」** 的写法一致（见下文第七节）；Python 里用列表存的 `input_vec` 当作列向量 **x** 使用即可。

### 三、权重矩阵怎么摆

书中约定（与图一致）：**矩阵的每一行对应一个输出神经元**，**每一列对应一个输入特征**（脚趾、胜负记录、粉丝数）。

```text
#           toes   win   fans
weights = [[ 0.1,  0.1, -0.3],   # 是否受伤
           [ 0.1,  0.2,  0.0],   # 是否胜利
           [ 0.0,  1.3,  0.1]]   # 是否难过
```

输入向量 `input_vec = [8.5, 0.65, 1.2]` 须与**每一行的列数**一致，才能与各行分别做点积。

---

### 四、分路计算（与图一致）

记 `x = [8.5, 0.65, 1.2]`。

| 输出     | 加权和 |
|----------|--------|
| 是否受伤 | 8.5×0.1 + 0.65×0.1 + 1.2×(−0.3) = 0.555 |
| 是否胜利 | 8.5×0.1 + 0.65×0.2 + 1.2×0 = 0.98 |
| 是否难过 | 8.5×0 + 0.65×1.3 + 1.2×0.1 = 0.965 |

---

### 五、完整可运行代码

```python
def w_sum(a, b):
    assert len(a) == len(b)
    output = 0
    for i in range(len(a)):
        output += a[i] * b[i]
    return output


def vect_mat_mul(vect, matrix):
    assert len(vect) == len(matrix[0])
    output = []
    for row in matrix:
        output.append(w_sum(vect, row))
    return output


def neural_network(input_vec, weights_matrix):
    pred = vect_mat_mul(input_vec, weights_matrix)
    return pred


weights = [
    [0.1, 0.1, -0.3],
    [0.1, 0.2, 0.0],
    [0.0, 1.3, 0.1],
]

toes = [8.5, 9.5, 9.9, 9.0]
wlrec = [0.65, 0.8, 0.8, 0.9]
nfans = [1.2, 1.3, 0.5, 1.0]

input_vec = [toes[0], wlrec[0], nfans[0]]

pred = neural_network(input_vec, weights)
print(pred)  # [0.555, 0.98, 0.965]
```

### 六、与前几种形态对照

| 形态 | 运算要点 |
|------|-----------|
| 多输入单输出 | 一行权重与输入点积，得到一个数 |
| 单输入多输出 | 标量与权重向量逐元素相乘，得到长度与权重相同的向量 |
| **多输入多输出** | **多行权重，每行与输入点积一次，得到长度等于行数的向量** |

---

### 七、矩阵乘法约定：与这段代码严格对应的是什么？

**结论（和本仓库 `vect_mat_mul` 一致）：** 遍历权重矩阵 **W 的每一行**，与**整个输入向量 x** 做点积，得到输出向量 **y** 的各个分量。

- 代码：`y = vect_mat_mul(x, W)`（参数名可叫 `input_vec`、`weights_matrix`）。
- 线性代数里常写成：**y = W · x**
  - **左边**：权重矩阵 **W**（行数 = 输出个数，列数 = 输入个数）
  - **右边**：输入 **列向量 x**（长度 = 输入个数）
  - **第 i 个输出** = **W 的第 i 行** 与 **x** 的点积，也就是「左矩阵的行 × 右向量的列」这一套规则。

**为什么有时会看到「输入在左」？** 批量训练时，常把**多条样本叠成矩阵 X**，每行是一条样本，于是出现 **Y = X · W′** 这类形式（要对 **W** 是否转置、哪一维是 batch 统一约定）。和 **y = W·x** 只是**摆放姿势不同**；对齐维度并正确使用转置后，**每个位置仍是「对应相乘再相加」的点积**，底层运算没变。

**本例尺寸（3 个输入、3 个输出）：** W 为 3×3，x 有 3 个分量；W 的**列数**（3）与 x 的**长度**（3）一致，才能按行做点积得到长度为 3 的 **y**。

---

### 八、小结

全连接一层（无激活）的前向传播，在「多输入多输出」情形下就是 **对权重矩阵的每一行做一次 `w_sum`**；`vect_mat_mul` 把这件事写清楚后，整层预测就是一行 `neural_network` 调用。与教科书 **W·x（权重在左、列向量在右）** 同构；框架里「样本行在左」的写法通过转置可与之一一对应。
