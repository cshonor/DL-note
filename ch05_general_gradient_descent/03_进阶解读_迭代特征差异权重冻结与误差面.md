# 多输入通用梯度下降 · 进阶解读：迭代、特征差异、冻结与误差面

> 对应主线：`02_多输入神经网络_图解详解_SGD与BGD.md`。本文专注进阶现象解释与实验观察。

---

## 一、三轮迭代怎么看

- 每一轮都执行同一闭环：`pred -> error/delta -> weight_deltas -> weights -= alpha * weight_deltas`。  
- 在 `input=[8.5, 0.65, 1.2]`、`weights=[0.1, 0.2, -0.1]`、`alpha=0.01` 下，前几轮误差通常下降较快。  
- 原因不是“跳到答案”，而是每轮沿当前局部斜率下山；越接近谷底，`delta` 越小，步幅自然变短。

---

## 二、为什么 a 特征那一路更新最猛

- 同一轮里 `delta` 对全部权重共享；区分各权重更新幅度的是 `x_i`。  
- `weight_delta_i = x_i * delta`，因此 `|x_i|` 越大，`|weight_delta_i|` 通常越大。  
- 本例 `8.5` 明显大于 `0.65` 和 `1.2`，所以对应权重更新更剧烈。  

这就是“特征尺度差异造成梯度不平衡”的直观来源。  
工程上常见处理：

1. 先做标准化/归一化，把特征量级拉近。  
2. 再调学习率 `alpha`，避免某一路权重震荡、主导训练。

---

## 三、单项权重冻结实验

脚本：`mult_input_train_loop.py` 已支持冻结/解冻。

冻结第一个权重 `w[0]`：

```bash
python ch05_general_gradient_descent/mult_input_train_loop.py --mode sgd0 --epochs 40 --every 5 --freeze-idx 0
```

第 25 轮解冻：

```bash
python ch05_general_gradient_descent/mult_input_train_loop.py --mode sgd0 --epochs 60 --every 10 --freeze-idx 0 --unfreeze-epoch 25
```

观察重点：

- 冻结时该维 `weight_delta` 被置零，因此该权重不更新。  
- 其他权重仍可更新，整体误差也可能继续下降。  
- 若系统已进入 `delta` 很小区间再解冻，被冻权重可能几乎不再动，表现为“误差信号太弱导致学习停滞”。

---

## 四、误差平面的本质

- 单看某个权重时，图上像一条 U 形切片。  
- 真实优化对象是高维误差面：3 个权重轴 + 1 个误差轴。  
- 梯度下降是在高维地形里沿局部最陡下降方向移动；单条 U 曲线只是某个视角下的截面投影。

---

## 五、和主线概念对齐

- `delta = pred - true`：节点端共享误差信号。  
- `weight_delta_i = x_i * delta`：把共享信号分配到每个权重。  
- 写成 `error = (pred-true)^2` 时会出现常数 2；常数可吸收到 `alpha`，不改变下降方向。  
- 深层网络中，`delta` 会继续乘更多局部导数向前传播，但“先有节点误差信号，再按输入分配到权重”这条骨架不变。
