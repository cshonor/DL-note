# 梯度不是给节点，而是给连接线

> 这篇只解决一个高频卡点：  
> “隐藏层一个节点接收多个输入时，反向传播为什么会有多个梯度？”

---

## 一、先把结构钉死

网络：`3 输入 -> 2 隐藏 -> 1 输出`

- 输入节点：`x1, x2, x3`（每个输入节点对应一个特征）
- 隐藏节点：`hA, hB`
- 输出：`y_hat`

前向：

- `hA = wA1*x1 + wA2*x2 + wA3*x3`
- `hB = wB1*x1 + wB2*x2 + wB3*x3`
- `y_hat = vA*hA + vB*hB`

---

## 二、最关键一句

**梯度不属于节点，梯度属于连接线（权重）。**

所以：

- 一个隐藏节点接收 3 条输入线  
- 就有 3 个权重（例如 `wA1,wA2,wA3`）  
- 就必须有 3 个独立梯度（`dL/dwA1,dL/dwA2,dL/dwA3`）

不是“一个节点一个梯度”，而是“一条边一个梯度”。

---

## 三、把 6 条边写全，就不会再晕

从输入到隐藏层有 6 条边：

- `x1 -> hA`（`wA1`）
- `x2 -> hA`（`wA2`）
- `x3 -> hA`（`wA3`）
- `x1 -> hB`（`wB1`）
- `x2 -> hB`（`wB2`）
- `x3 -> hB`（`wB3`）

每条边都要单独算梯度。

---

## 四、按链式法则一步推到位（线性层版本）

设误差项写作：

- `e = y_hat - y`

先看输出层两条边：

- `dL/dvA = e * hA`（若用无 `0.5` 平方误差，整体还会有常数 2，通常并入学习率）
- `dL/dvB = e * hB`

再往前到隐藏层输入边：

- 传给 `hA` 这一路的因子：`e * vA`
- 传给 `hB` 这一路的因子：`e * vB`

于是：

- `dL/dwA1 = (e * vA) * x1`
- `dL/dwA2 = (e * vA) * x2`
- `dL/dwA3 = (e * vA) * x3`
- `dL/dwB1 = (e * vB) * x1`
- `dL/dwB2 = (e * vB) * x2`
- `dL/dwB3 = (e * vB) * x3`

你会看到：

- 同一隐藏节点内，三条边共享“同一个后向因子”（`e*vA` 或 `e*vB`）
- 但最终梯度仍因输入 `x1,x2,x3` 不同而分成三条

---

## 五、一句话打通

反向传播做的事就是：

1. 从最终误差出发  
2. 乘后续路径的局部导数（线性层里常表现为后层权重）  
3. 乘这条边起点的输入  
4. 得到“这条边”的梯度

所以层再深、节点再多，规则都不变：

**一条边一个权重，一条边一个梯度。**
