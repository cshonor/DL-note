# 每条连接线一个梯度：矩阵存储与独立更新

> 这篇只讲工程落地版结论：梯度到底挂在哪、怎么更新、线与线之间会不会互相干扰。

---

## 一、先拍板：每条线一个梯度

以 `3 -> 2 -> 1` 为例：

- 输入层 3 个节点
- 隐藏层 2 个节点
- 输入到隐藏是全连接：`3 x 2 = 6` 条边

因此一定有：

- 6 个权重
- 6 个对应梯度

结论：

**梯度属于连接线（权重参数），不属于节点。**

---

## 二、更新规则永远不变

每条线只按自己的梯度更新：

- `w_new = w_old - lr * grad`

例子：

- `w=0.5`, `grad=0.2`, `lr=0.1`
- 更新后：`0.5 - 0.1*0.2 = 0.48`

---

## 三、一条线更新会影响别的线吗

在**参数更新这一步**，不会。

- 每个参数用自己的梯度独立更新
- 不需要“等另一条线更新完再更新这条线”

这也是为什么可以并行化：

- 参数张量和梯度张量同形状
- 对应元素做同一条更新公式

---

## 四、梯度存在哪里：矩阵，不绑节点

`3 -> 2` 的权重可写成 `2x3` 矩阵（按“每个隐藏节点一行”）：

```text
W =
[
  [w11, w12, w13],   # hidden1 <- x1,x2,x3
  [w21, w22, w23]    # hidden2 <- x1,x2,x3
]
```

梯度是同形状矩阵：

```text
dW =
[
  [dw11, dw12, dw13],
  [dw21, dw22, dw23]
]
```

更新就是对应位置相减：

- `W = W - lr * dW`

这就是框架内部的核心操作形态。

---

## 五、一个隐藏节点接 3 个输入，梯度怎么分到 3 条线

关键规律：

- 同一个隐藏节点对应的几条入边，通常共享同一个“后向误差因子”
- 但每条边还要乘自己的输入值

例如对隐藏节点 `h1`：

- 公共因子：`factor_h1`（由后层误差反传得到）
- 三条边梯度：
  - `dw11 = factor_h1 * x1`
  - `dw12 = factor_h1 * x2`
  - `dw13 = factor_h1 * x3`

所以：

- 因子相同
- 输入不同
- 梯度自然不同

---

## 六、六句通关总结

1. 每条连接线一个权重，也对应一个梯度。  
2. 梯度存储在和权重同形状的张量/矩阵里。  
3. 更新规则统一：`param -= lr * grad`。  
4. 更新阶段各参数独立，不互相“抢更新”。  
5. 前层梯度来自“后向因子 × 本边输入”。  
6. 多层网络只是重复同一规则，形状更大而已。
