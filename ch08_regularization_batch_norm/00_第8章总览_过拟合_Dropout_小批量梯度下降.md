# 第8章：过拟合、Dropout 与小批量梯度下降
**Core Concept: Overfitting -> Regularization with Dropout -> Stable Training with Mini-batch**

## 一、整章一句话
第8章就是训练实战三连：  
**模型为什么会学太死（过拟合） -> 怎么压住（Dropout） -> 怎么训得又快又稳（Mini-batch）。**

---

## 二、本章只讲三件关键事

### 1) 过拟合（Overfitting）
- **人话解释**：模型把训练集里的噪声和巧合也记住了。
- **本质**：训练表现很好，但泛化能力变差。
- **典型表现**：训练准确率高，测试准确率低；训练 loss 降，验证 loss 反升。

一句话：  
**不是模型不会学，而是学得太死。**

---

### 2) Dropout（经典防过拟合手段）
- **做法**：训练阶段随机关闭一部分神经元。
- **目的**：防止模型过度依赖少数节点，减少“背答案”行为。
- **效果**：特征学习更分散、更稳健，泛化通常更好。
- **使用规则**：训练时开 Dropout，推理时关 Dropout。

工程直觉：  
**Dropout 相当于强制模型别走单一路径，逼它学可迁移的规律。**

---

### 3) 易混淆点澄清：为什么“固定冻结”不推荐，但 Dropout 推荐

这个问题非常关键，核心在于两者的“冻结机制”完全不同。

#### 普通权重冻结（不鼓励）是什么
- 人为指定某些层或某些权重长期不更新。
- 冻结位置固定、持续时间长、通常全程不变。
- 结果是模型搜索空间被锁死，容易卡在局部解。

常见风险：
- 全局协同优化能力下降。
- 过度依赖少数可训练参数。
- 更容易出现僵化与过拟合。

#### Dropout 的“冻结”（鼓励）是什么
- 每次前向都随机屏蔽一部分神经元（临时失活）。
- 每个 batch 的屏蔽集合都在变化，不固定、不长期绑定。
- 推理阶段关闭 Dropout，使用完整网络。

它的本质不是“锁死参数”，而是“打散依赖关系”：
- 防止某些神经元长期抱团主导。
- 迫使模型学习更通用、更鲁棒的特征组合。

一句话对比：
- **固定冻结**：固定结构 + 固定依赖 -> 容易锁进局部路径。
- **Dropout**：随机扰动 + 随机打散依赖 -> 降低共适应、提升泛化。

---

### 4) L2 正则为什么会把权重往 0 拉
先看更新公式（梯度下降）：

`w_j = w_j - eta * dLoss/dw_j`

如果损失写成：

`Loss = 误差项 + lambda * sum(w_j^2)`

那么对 `w_j` 求导会多出一项：

`dLoss/dw_j = d(误差)/dw_j + 2 * lambda * w_j`

代回更新式：

`w_j = w_j - eta * d(误差)/dw_j - eta * 2 * lambda * w_j`

关键就在最后一项：

`- eta * 2 * lambda * w_j`

- 当 `w_j > 0`，这一项是负数，`w_j` 会减小，向 0 靠。
- 当 `w_j < 0`，这一项相当于加一个正数，`w_j` 会增大，也向 0 靠。

所以不管正负，L2 每一步都在把权重轻轻往 0 拉，这就是权重衰减（weight decay）的直觉来源。

补充：有的教材写成 `(lambda/2) * sum(w_j^2)`，这时导数是 `lambda * w_j`。  
系数不同只是记号选择，不改变“往 0 拉”的本质。

---

### 5) L1 与 L2 正则化对比（只盯“怎么拉权重”）

先写 L1：

`Loss = 误差项 + lambda * sum(|w_j|)`

对 `w_j` 求导时，正则项贡献是符号函数：

- `w_j > 0` -> `+1`
- `w_j < 0` -> `-1`
- `w_j = 0` 处严格不可导，工程上用次梯度处理

所以更新里多出的项是：

`- eta * lambda * sign(w_j)`

这项的作用：

- 若 `w_j > 0`，每步都减一个固定量 `eta * lambda`，往 0 拉。
- 若 `w_j < 0`，每步都加一个固定量 `eta * lambda`，也往 0 拉。

关键区别（和 L2 对照）：

- **L2**：`- eta * 2 * lambda * w_j`
  - 拉力和 `|w_j|` 成正比（大权重大拉，小权重小拉）
  - 更平滑，通常把权重压小但不一定压成精确 0

- **L1**：`- eta * lambda * sign(w_j)`
  - 拉力近似固定，不看当前权重大小
  - 更容易把小权重直接推到 0，形成稀疏性（自动特征选择）

一句话收束：

- `L2`：温柔收缩，权重普遍变小  
- `L1`：强硬稀疏，不重要权重更容易归零

---

### 6) 批量梯度下降方式（Batch / SGD / Mini-batch）
核心问题是：**一次用多少样本更新权重**。

- **Batch Gradient Descent**
  - 用全量训练集算一次梯度再更新
  - 梯度稳定，但速度慢、资源压力大

- **SGD（随机梯度下降）**
  - 每次只用 1 个样本更新
  - 更新频繁、速度快，但抖动大

- **Mini-batch Gradient Descent（主流）**
  - 每次用一小批样本（常见 32 / 64 / 128）
  - 在稳定性、速度、显存占用之间取得最好平衡

本章实践结论：  
**小批量训练是现代深度学习默认方案。**

## 四、本章核心结论（收束）
- 过拟合是训练实战中最常见风险之一。
- Dropout 是简单有效、成本低的防过拟合工具。
- Mini-batch 是训练效率和稳定性的工程最优折中方案。
- 这三者合在一起，构成“能训、稳训、泛化不崩”的基础框架。
