# NLP 最小完整流程：从随机词向量到可用分类

> 目标：只用你已经会的四块（词向量、嵌入层、池化、全连接）讲清 NLP 从头到尾怎么跑。

---

## 一、先回答最关键问题
**词向量一开始是随机的。**

你只定义两件事：
1. `vocab_size`：词表大小  
2. `embed_dim`：词向量维度

模型会创建一个参数矩阵：

`[vocab_size, embed_dim]`

初始值通常是随机小数，语义关系并不存在，是训练后学出来的。

---

## 二、最小 NLP 流程（不含 RNN）

### 第 0 步：准备
1. 建词表并分配 ID  
2. 设定嵌入维度  
3. 初始化嵌入矩阵（随机）

---

### 第 1 步：句子转 ID
例如：

`我 喜欢 苹果 -> [1, 2, 3]`

---

### 第 2 步：过嵌入层（查表）
输入 ID 后，从嵌入矩阵按行取向量：
- 词1 -> 100维
- 词2 -> 100维
- 词3 -> 100维

得到词向量序列，形状可看作：

`(N, embed_dim)`，例如 `(3, 100)`

---

### 第 3 步：池化/聚合
把多个词向量汇总成一个句子向量（如平均池化）：

`(N, embed_dim) -> (1, embed_dim)`

例如：

`(3, 100) -> (1, 100)`

---

### 第 4 步：全连接分类
句子向量输入分类头（例如 `100 -> 2`），输出类别分数（如正面/负面）。

---

### 第 5 步：误差反传与更新
计算损失后反向传播，更新两部分参数：
1. 全连接层权重  
2. 嵌入矩阵参数

这一步会让词向量从“随机”逐渐变成“有语义结构”。

---

## 三、“词向量是不是自定义”这件事
不是手工定义语义含义。  
你只定义结构（词表大小、维度），语义由模型训练学出来。

训练前：
- 向量几乎无语义

训练后：
- 相近语义词更靠近
- 可形成稳定的语义空间结构

---

## 四、为什么说全连接不擅长句子建模
全连接流程能跑通分类任务，但有明显限制：
- 对词序建模能力弱
- 单纯池化会压缩顺序信息

这就是后续要引入 RNN/LSTM/Transformer 的动机。  
但它们是在“嵌入后处理中”增强能力，不是否定基础流程。

---

## 五、最短背诵版
**NLP 最小流程就是：句子转 ID -> 嵌入查表 -> 池化成句向量 -> 全连接分类 -> 反向传播更新（含嵌入矩阵），最终词向量从随机变成有语义。**
