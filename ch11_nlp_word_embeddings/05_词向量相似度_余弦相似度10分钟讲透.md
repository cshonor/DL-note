# 词向量相似度：余弦相似度 10 分钟讲透

## 一、它是干嘛的（一句话）
**词向量相似度就是用来判断两个词或两句话“意思像不像”。**

- 越接近 `1`：越像  
- 越接近 `0`：越不像  
- 接近 `-1`：方向相反（在部分任务里会映射到 `0~1`）

---

## 二、最常用方法：余弦相似度（Cosine Similarity）
在 NLP / RAG 里最常用的就是余弦相似度。

核心思想很简单：
- 不重点看向量长度
- 主要看向量方向是否一致

方向越一致，语义通常越接近。

---

## 三、点积和余弦相似度到底什么关系
你提到的这个关系式就是核心：

`A·B = |A| * |B| * cos(theta)`

其中：
- `A·B` 是点积（内积）
- `|A|`、`|B|` 是向量长度
- `cos(theta)` 就是两个向量夹角的余弦

把它变形就得到余弦相似度：

`cos(theta) = (A·B) / (|A| * |B|)`

这一步的意义是：  
**用点积做分子，再把“长度影响”除掉，只保留方向相似性。**

---

## 四、最直观小例子（2 维）
假设有三个向量：
- 男人：`(1, 2)`
- 女人：`(1, 3)`
- 苹果：`(5, -1)`

直观上：
- 男人和女人方向接近 -> 相似度会更高
- 男人和苹果方向差别大 -> 相似度会更低

---

## 五、记住 3 条就够
1. 余弦相似度常见范围：`-1 ~ 1`  
2. 越接近 `1`，语义越相近  
3. 越接近 `0`，语义相关性越弱

在检索场景里通常做法是：
- 对候选向量算相似度
- 按分数从高到低排序
- 取 Top-K

---

## 六、它在 NLP / RAG 链路里的位置
你现在整条链路是：

1. 文本 -> 词 ID  
2. 嵌入层 -> 词向量序列  
3. 池化（聚合）-> 句向量  
4. 余弦相似度 -> 判断句子是否接近  
5. 用于检索 / RAG / 匹配任务

---

你已经掌握的四块核心：
1. 词向量

这四块组合起来，就是 NLP 与 RAG 的核心基础原理。
