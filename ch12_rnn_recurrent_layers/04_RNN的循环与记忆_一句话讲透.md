# RNN 的循环与记忆：一句话讲透

## 一句话终极总结
RNN 的循环 = **同一套权重反复按时间步调用**  
RNN 的记忆 = **把上一步隐藏状态 `h_{t-1}` 作为当前输入之一继续参与计算**

---

## 1. RNN 是怎么“循环”的（结构层面）
普通全连接层：
- 输入 -> 计算 -> 输出
- 这一轮结束就结束，不会把结果喂回自己

RNN：
1. 固定一套权重（输入权重 `U`，循环权重 `W`）
2. 维护一个“上一时刻状态”向量 `h_prev`
3. 每个时间步都执行同一个公式：

`h_t = tanh(W @ h_prev + U @ x_t)`

4. 计算完把 `h_t` 保存为下一步的 `h_prev`

这就是循环：
- 不是不断新增层
- 不是不断换权重
- 而是“同一层、同一组参数”沿时间重复使用

---

## 2. RNN 的“记忆”到底是什么
记忆并不神秘，就是：

**上一时刻隐藏层输出的整条向量 `h_prev`**

它不是：
- 额外神秘缓存
- 权重参数
- 梯度

它就是上一步算出来的隐藏状态数值。

---

## 3. 多隐藏节点时怎么理解
如果隐藏层有 128 个单元：
- `h_prev` 就是一个长度 128 的向量
- 每个位置对应一个隐藏单元上一步的输出
- 当前时刻计算时，整条 128 维向量一起参与

所以 RNN 记忆不是一个数，而是“一整层状态”。

---

## 4. RNN 与全连接的核心差异
- 全连接：只看当前输入，不显式引入过去状态  
- RNN：当前输入 `x_t` + 过去状态 `h_{t-1}` 一起决定当前状态  
- 全连接层间参数通常各自独立  
- RNN 在时间步上共享同一套参数 `U/W`

---

## 5. 最简流程（可直接背）
1. 初始化 `h0 = 0` 向量  
2. 输入 `x1`，算 `h1 = tanh(W@h0 + U@x1)`  
3. 输入 `x2`，算 `h2 = tanh(W@h1 + U@x2)`  
4. 输入 `x3`，算 `h3 = tanh(W@h2 + U@x3)`  
5. 持续重复

记忆就是“上一时刻的 h”，循环就是“重复用同一套 `W/U`”。

---

## 最终刻进脑子里的两句话
1. **RNN 的循环 = 同一层在时间轴上反复调用。**  
2. **RNN 的记忆 = 上一步隐藏状态继续作为下一步输入。**
