# 手算 RNN 求导：为什么 W 连乘导致梯度消失

这篇只做一件事：  
不用矩阵、不用复杂符号，只用可手算的方式看懂：
**为什么对 RNN 的循环权重 `W` 求导，会出现连乘，最后导致梯度消失。**

---

## 1) 最简化设定

为了只看本质，先做三件简化：
- 去掉激活函数（只看线性递推）
- 去掉偏置
- 把所有变量都当成实数

最简 RNN 递推写成：

`h_t = W * h_(t-1) + U * x_t`

这里：
- `W` 是循环权重（我们要求导的参数）
- `U * x_t` 是当前输入项

---

## 2) 先展开 3 个时间步（俄罗斯套娃）

`h_1 = W * h_0 + U * x_1`  
`h_2 = W * h_1 + U * x_2`  
`h_3 = W * h_2 + U * x_3`

把 `h_3` 往里展开：

`h_3 = W * (W * (W * h_0 + U * x_1) + U * x_2) + U * x_3`

这就是“套娃”：里面层层都有 `W`。

---

## 3) 手算 `d h_3 / dW`

从外到内，用乘法求导：

`d h_3 / dW = d(W * h_2 + U * x_3) / dW`  
`= h_2 + W * d h_2 / dW`

继续展开 `d h_2 / dW`：

`d h_2 / dW = d(W * h_1 + U * x_2) / dW`  
`= h_1 + W * d h_1 / dW`

再展开 `d h_1 / dW`：

`d h_1 / dW = d(W * h_0 + U * x_1) / dW`  
`= h_0`  （把 `h_0` 看成常量）

代回去得到：

`d h_3 / dW = h_2 + W * h_1 + W^2 * h_0`

---

## 4) 这一步已经看到关键规律

`d h_3 / dW = h_2 + W * h_1 + W^2 * h_0`

解释：
- 越近的时刻，乘的 `W` 越少
- 越早的时刻，乘的 `W` 次数越多

也就是：**历史越久远，梯度项被更多次 `W` 缩放。**

---

## 5) 推广到 T 步

同样规律可写成：

`d h_T / dW = h_(T-1) + W * h_(T-2) + W^2 * h_(T-3) + ... + W^(T-1) * h_0`

如果 `|W| < 1`，那 `W^(T-1)` 会随着 `T` 变大快速趋近 0。  
于是最早时刻对应的梯度项几乎消失。

这就是基础 RNN 的梯度消失直观来源。

---

## 6) 为什么对 U 的感觉不一样

同样看 3 步，`U` 对当前步是直接项：  
例如 `h_3` 里直接有 `U * x_3`，对 `U` 求导立刻出现 `x_3`。

虽然更早时刻的 `U` 贡献也会通过链式传递，但最直观上，`W` 的“反复自乘”问题最突出，所以梯度消失常先从 `W` 的连乘角度解释。

---

## 最简背诵版

1. RNN 时间展开后是嵌套结构（套娃）  
2. 对循环权重 `W` 求导会得到按时间求和的多项式  
3. 越早时刻对应越高次的 `W` 幂  
4. 当 `|W| < 1`，高次幂快速衰减到 0  
5. 结果：早期信息梯度几乎传不回去（梯度消失）
